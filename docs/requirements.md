# 요구사항 정의서

## 기능 요구사항 (Functional Requirements)
|ID|요구사항 내용|우선순위|
|----|--------------|--------|
|FR-01|사용자는 정보의 공유를 원할 때 데이터 내보내기에서 이미지 저장(캡처 ui)이 가능해야 한다|하|
|FR-02|시스템은 선수 비교 기능 중 어떤 항목(타율, 구단 등 능력치)에 따라 비교할지 정의될 수 있어야 한다|상|
|FR-03|시스템은 선수 비교 기능 중에서 비교 화면을 구현(그래프, 카드 ui)할 수 있어야 한다|상|
|FR-04|시스템은 필터 기능에서 상황별 데이터 구조 설계가 되어 있어야 한다|상|
|FR-05|시스템은 필터 기능 중 사용자에게 유용한 필터만 남겨질 수 있어야 한다|중|


## 비기능 요구사항 (Non-Functional Requirements)
|ID|요구사항 내용|우선순위|
|----|--------------|---------|
|NFR-01|선수별 데이터 업데이트 속도는 경기가 끝난 후 3시간 이내에 결과를 반영할 수 있어야 한다 (성능)|상|
|NFR-02|초보자도 사용할 수 있게끔 인터페이스는 단순하며 핵심 정보를 담고 있어야 한다 (사용성)|상|
