## 논의 내용

### 1. 설계 패턴 정립
- 이전 회의에서 작성한 클래스 다이어그램(`FilterManager`, `ExportService`, `PlayerService`)을 바탕으로 적용할 패턴 논의
- `FilterManager`와 `ExportService`는 전역에서 단일 인스턴스를 유지해야 한다는 점에서 **싱글톤 패턴** 적용으로 결정
- 사용자 모드(입문자/일반 팬/분석가)에 따라 렌더링 방식이 달라지므로 `PlayerService`에 **전략 패턴** 적용으로 결정

### 2. SOLID 원칙 검토
- SRP: `PlayerService`와 `ExportService`가 역할을 분리하고 있어 준수로 확인
- OCP: `FilterManager`가 추상 구조를 바라보도록 설계되어 있어 준수로 확인
- LSP: `Admin`이 `User`를 상속하면서 기존 메서드를 무력화하지 않으므로 준수로 확인
